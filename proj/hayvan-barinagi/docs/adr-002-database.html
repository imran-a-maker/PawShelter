Bağlam (Context)

PawShelter platformunun veri modeli aşağıdaki ilişkisel yapıları içermektedir:

  • Kullanıcılar (Users) ↔ Sahiplenme Talepleri (AdoptionRequests): 1-N ilişki
  • Hayvanlar (Animals) ↔ Sahiplenme Talepleri: 1-N ilişki
  • Hayvanlar ↔ Sağlık Kayıtları (HealthRecords): 1-N ilişki
  • Kullanıcılar ↔ Roller (Roles): N-N ilişki

Değerlendirilen veritabanı alternatifleri:

  1. PostgreSQL (İlişkisel, ACID uyumlu) ← Seçilen
  2. MongoDB (Döküman tabanlı, NoSQL)
  3. MySQL / MariaDB (İlişkisel)
  4. SQLite (Geliştirme ortamı için)

Karar (Decision)

PostgreSQL birincil veritabanı olarak seçilmiştir. Geliştirme ortamında SQLite kullanılabilir (SQLAlchemy soyutlaması sayesinde).

Kriter PostgreSQL ✅ MongoDB
ACID Uyumluluğu ✅ Tam destek ⚠️ Kısmi (v4.0+)
İlişkisel Veri ✅ Native JOIN ❌ $lookup (yavaş)
JSON Desteği ✅ JSONB tipi ✅ Native
Şema Esnekliği ⚠️ Migration gerekli ✅ Schemaless
Veri Bütünlüğü ✅ FK Constraints ❌ Uygulama seviyesi
Full-Text Search ✅ tsvector ✅ Atlas Search
Ücretsiz & Açık Kaynak ✅ Tamamen ücretsiz ⚠️ Community Edition

Sonuçlar (Consequences)

✅ Avantajlar

  • Sahiplenme işlemlerinde ACID garantisi ile veri tutarlılığı sağlanır
  • Foreign key constraint'ler ile referans bütünlüğü otomatik korunur
  • Alembic ile migration yönetimi kolaylaşır
  • JSONB tipi ile esnek veri depolama imkânı
  • Ücretsiz, açık kaynak, geniş topluluk desteği

❌ Dezavantajlar

  • Şema değişikliklerinde migration yazılması gerekir
  • Yatay ölçekleme MongoDB'ye göre daha karmaşık
  • Çok büyük ölçekte sharding yapılandırması gerekebilir
